# HajaCheck 착수보고 슬라이드 디자인 가이드

기준 이미지: `C:\HajaCheck_report\data\example\1.png` ~ `8.png`  
작성 목적: 예시 템플릿의 시각 언어를 유지하면서, 신규 착수보고 슬라이드를 일관되게 제작하기 위한 상세 디자인 가이드

---

## 1. 전체 디자인 컨셉

### 1.1 핵심 컨셉

HajaCheck 착수보고 슬라이드는 **"신뢰 가능한 AI 점검 플랫폼의 초기 설계 보고서"**라는 인상을 주어야 한다.  
디자인은 화려한 홍보물보다는 **정리된 컨설팅 보고서**, **기술 프로젝트 착수 문서**, **점검/검증 플랫폼 제안서**에 가깝게 제작한다.

핵심 키워드는 다음과 같다.

- **Clean**: 불필요한 장식 없이 넓은 여백과 명확한 정보 구조를 사용한다.
- **Reliable**: 검정 텍스트, 흰색 패널, 안정적인 좌우 정렬로 신뢰감을 만든다.
- **Technical but Friendly**: 기술 보고서이지만 파란색 아이콘, 둥근 카드, 부드러운 배경색으로 접근성을 높인다.
- **Check & Detection**: 체크, 탐지, 검색, 보고서, 데이터, 흐름을 상징하는 아이콘을 반복해 서비스 성격을 강화한다.
- **Structured Progress**: 섹션 번호와 콘텐츠 패널을 통해 프로젝트가 단계적으로 정리되어 있다는 느낌을 준다.

### 1.2 시각적 인상

전체 톤은 **밝고, 차분하고, 정돈된 전문성**이다.  
배경은 매우 밝은 회색, 주요 콘텐츠는 흰색 라운드 패널, 강조는 선명한 파란색으로 구성한다.  
무거운 기술 주제라도 화면은 답답해 보이면 안 된다. 정보는 많더라도 여백을 통해 숨 쉴 공간을 확보해야 한다.

권장 인상:

- "프로젝트가 이미 체계적으로 정리되어 있다"
- "AI 기반 서비스지만 과장되지 않고 현실적인 수행 계획을 갖고 있다"
- "보고서, 점검, 검증, 산출물 관리가 명확하다"
- "팀이 발표자료를 급하게 만든 것이 아니라 하나의 시스템처럼 설계했다"

피해야 할 인상:

- 스타트업 랜딩페이지처럼 과하게 홍보성인 느낌
- 복잡한 대시보드처럼 정보가 과밀한 느낌
- 형광색, 그라데이션, 그림자 과다 사용으로 가벼워 보이는 느낌
- 아이콘과 도형이 제각각이라 템플릿 통일성이 사라지는 느낌

## 2. 캔버스와 기본 레이아웃

### 2.1 슬라이드 규격

- 비율: **16:9 와이드**
- 권장 픽셀 기준: **1920 x 1080**
- 작업 기준: PowerPoint, Keynote, Figma, Canva 모두 동일하게 16:9로 제작한다.
- 예시 이미지 기준 모든 장표가 1920 x 1080이므로, 이 비율을 변경하지 않는다.

### 2.2 기본 배경

모든 본문 장표의 배경은 흰색이 아니라 **아주 밝은 회색**을 사용한다.

- 권장 배경색: `#F5F5F5` 또는 `#F6F6F6`
- 목적: 흰색 콘텐츠 패널이 배경 위에서 자연스럽게 구분되게 한다.
- 배경에는 패턴, 이미지, 그라데이션, 장식 도형을 넣지 않는다.

커버와 목차는 본문 장표보다 더 많은 여백을 사용하며, 동일한 밝은 회색 배경을 유지한다.

### 2.3 안전 여백

슬라이드 바깥쪽 여백은 넉넉하게 유지한다.

- 좌측 기본 여백: 약 **115px**
- 우측 기본 여백: 약 **115px**
- 상단 제목 영역: 약 **105px ~ 170px**
- 하단 푸터 영역: 약 **990px 부근**
- 본문 콘텐츠 패널 상단: 약 **200px**
- 본문 콘텐츠 패널 하단: 약 **960px**

상단 제목, 우측 문서명, 하단 푸터와 페이지 번호는 고정 요소처럼 다룬다.  
본문 콘텐츠는 이 고정 요소와 겹치지 않아야 한다.

### 2.4 본문 패널

본문 장표 대부분은 큰 흰색 라운드 패널을 중심으로 구성한다.

- 위치: 좌우 약 115px 여백 안쪽
- 크기: 약 **1690px x 760px**
- 상단 위치: 약 200px
- 하단 위치: 약 960px
- 배경색: `#FFFFFF`
- 모서리 반경: **36px ~ 44px**
- 그림자: 사용하지 않거나 거의 보이지 않게 한다.
- 테두리: 기본적으로 사용하지 않는다.

패널은 정보를 담는 그릇이다.  
패널 안에 또 큰 카드형 섹션을 중첩하면 답답해지므로, 내부 카드는 필요할 때만 작게 사용한다.

## 3. 색상 시스템

### 3.1 메인 컬러

예시 템플릿의 핵심 색상은 선명한 파란색이며, **포인트 컬러는 반드시 `#509BF8`로 유지한다.**

- 메인 블루 / 포인트 컬러: `#509BF8`
- 강조 블루: `#509BF8`
- 라이트 블루 배경: `#EAF4FF` 또는 `#E8F2FF`
- 연한 연결선 블루: `#9FC4EF`
- 차트 보조 블루: `#BFD9FB`

메인 블루 `#509BF8`은 다음 요소에 사용한다.

- 섹션 번호
- 핵심 키워드
- 체크된 칩
- 주요 아이콘
- 차트 막대
- 강조 라벨
- 목차의 큰 번호

### 3.2 텍스트 컬러

- 주요 제목: `#1A1A1A`
- 본문 텍스트: `#222222`
- 보조 텍스트: `#666666` ~ `#777777`
- 푸터 텍스트: `#A0A0A0`
- 라인/구분선: `#D8D8D8`, 진한 구분선은 `#222222`

검정 텍스트는 완전한 `#000000`보다 약간 부드러운 검정 계열을 권장한다.  
단, 큰 제목은 강한 대비가 필요하므로 `#161616` ~ `#1A1A1A`를 사용한다.

### 3.3 브랜드 보조색

HajaCheck 로고에 오렌지 포인트가 있으나, 슬라이드 본문에서는 오렌지를 적극 사용하지 않는다.  
오렌지는 로고 안에서만 보이는 정도가 적절하다.

권장:

- 로고 원본 색상 유지
- 본문 강조는 `#509BF8` 파란색으로 통일
- 오렌지, 초록, 빨강 등 다른 강조색은 상태 표현이 꼭 필요할 때만 제한적으로 사용

금지에 가까운 사용:

- 파란색과 오렌지를 같은 비중으로 사용하는 것
- 그라데이션 배경으로 파란색을 과하게 확장하는 것
- KPI마다 서로 다른 원색을 부여하는 것

## 4. 타이포그래피

### 4.1 권장 폰트

폰트는 `C:\HajaCheck_report\data\font` 폴더에 포함된 **에이투지체**를 기본으로 사용한다.  
슬라이드 전체의 제목, 본문, 숫자, 캡션은 모두 에이투지체 계열 안에서 굵기만 달리해 통일한다.

- 기본 한글/영문: **에이투지체**
- 제목/섹션 번호/큰 숫자: **에이투지체-8ExtraBold** 또는 **에이투지체-9Black**
- 소제목/카드 제목: **에이투지체-6SemiBold** 또는 **에이투지체-7Bold**
- 본문: **에이투지체-4Regular** 또는 **에이투지체-5Medium**
- 보조 문구/푸터: **에이투지체-3Light** 또는 **에이투지체-4Regular**
- 로고/브랜드명: 로고 이미지 또는 브랜드 지정 서체 사용

에이투지체는 굵기 단계가 세분화되어 있어, 예시 템플릿처럼 큰 제목은 강하게, 본문은 담백하게, 푸터는 가볍게 처리하기 좋다.  
다른 폰트를 섞으면 장표 간 인상이 흔들리므로, 특별한 사유가 없는 한 에이투지체만 사용한다.

### 4.2 글자 굵기

- 커버 타이틀: Black 또는 ExtraBold
- 섹션 제목: ExtraBold 또는 Bold
- 본문 소제목: Bold
- 본문 설명: Regular 또는 Medium
- 보조 문구: Regular
- 숫자 강조: ExtraBold 또는 Black

전체적으로 제목은 강하고, 본문은 담백해야 한다.  
모든 텍스트를 Bold로 만들면 정보 위계가 무너진다.

### 4.3 글자 크기 기준

1920 x 1080 기준 권장 크기:

- 커버 메인 타이틀 `HajaCheck`: 약 **86px ~ 96px**
- 커버 상단 설명: 약 **24px ~ 28px**
- 커버 부제 `착수보고서`: 약 **28px ~ 32px**
- 본문 섹션 번호: 약 **58px ~ 68px**
- 본문 장표 제목: 약 **50px ~ 60px**
- 우측 상단 문서명: 약 **24px ~ 28px**
- 패널 안 핵심 문장: 약 **34px ~ 42px**
- 카드 제목: 약 **28px ~ 34px**
- 본문 문장: 약 **22px ~ 28px**
- 작은 설명: 약 **16px ~ 20px**
- 푸터/페이지 번호: 약 **20px ~ 24px**

실제 제작 시 텍스트가 길어지면 폰트 크기를 무리하게 줄이기보다 문장을 줄인다.  
이 템플릿은 "많이 쓰는" 디자인이 아니라 "핵심만 크게 보여주는" 디자인이다.

### 4.4 줄 간격과 문장 길이

- 제목 줄 간격: 1.05 ~ 1.15
- 본문 줄 간격: 1.35 ~ 1.55
- 카드 안 텍스트: 2줄 이내 권장
- 긴 설명 문단: 3줄을 넘기지 않는다.

강조 문장은 가운데 정렬을 사용해도 되지만, 긴 본문은 반드시 좌측 정렬한다.  
한 줄의 글자 수가 너무 길면 읽기 어렵기 때문에, 본문 영역은 적절한 너비로 제한한다.

## 5. 공통 구성 요소

### 5.1 상단 섹션 헤더

본문 장표의 헤더는 다음 구조를 따른다.

```text
좌측: 01  프로젝트 소개
우측: HajaCheck 착수보고
```

제작 규칙:

- 섹션 번호는 파란색, 제목은 검정색으로 둔다.
- 번호와 제목은 같은 기준선에 맞춘다.
- 번호와 제목 사이 간격은 약 **18px ~ 28px**로 둔다.
- 제목은 매우 굵게 처리한다.
- 우측 상단 문서명은 작고 굵게 두되, 로고보다 텍스트 문서명에 가깝게 처리한다.

섹션 번호는 단순한 페이지 번호가 아니라 장표의 장(chapter)을 나타낸다.  
따라서 `01`, `02`, `03`처럼 두 자리 숫자를 유지한다.

### 5.2 하단 푸터

본문 장표 하단에는 좌측 보고서명, 우측 페이지 번호를 배치한다.

```text
좌측: 인공지능개발 양성과정_킥오프 보고서
우측: 3
```

제작 규칙:

- 푸터 텍스트 색상은 연한 회색으로 둔다.
- 페이지 번호도 같은 톤으로 처리한다.
- 본문보다 존재감이 낮아야 한다.
- 푸터는 장표마다 위치가 흔들리면 안 된다.

### 5.3 흰색 라운드 패널

가장 중요한 본문 컨테이너다.

사용 목적:

- 정보 묶음의 배경
- 도표, 카드, 아이콘, 문장 등을 담는 무대
- 배경과 콘텐츠를 부드럽게 분리

제작 규칙:

- 항상 넓게 사용한다.
- 패널 안쪽 여백은 최소 **90px 이상** 확보한다.
- 패널 안 요소가 모서리에 너무 붙으면 템플릿의 고급스러운 느낌이 사라진다.
- 패널 내부에 많은 선을 넣지 않는다.

### 5.4 라이트 블루 카드

`3.png`, `7.png`에서 보이는 연한 파란색 카드는 요약 정보나 키워드 묶음에 사용한다.

제작 규칙:

- 배경색: `#EAF4FF`
- 모서리 반경: **14px ~ 24px**
- 내부 여백: **36px ~ 48px**
- 카드 간 간격: **30px ~ 42px**
- 텍스트는 좌상단 배치
- 아이콘은 우하단 배치

카드에는 하나의 메시지만 넣는다.  
카드 하나에 제목, 설명, 숫자, 아이콘, 주석을 모두 넣으면 단순한 템플릿의 장점이 사라진다.

### 5.5 파란색 칩

`6.png`에서 사용된 칩은 키워드, 태그, 체크 항목을 나타내는 요소다.

종류:

- 비활성 칩: 흰색 배경 + 회색 테두리 + 회색 텍스트
- 활성 칩: 파란색 배경 + 흰색 체크 아이콘 + 흰색 텍스트

제작 규칙:

- 높이: 약 **80px ~ 100px**
- 좌우 패딩: 약 **40px ~ 60px**
- 모서리 반경: 높이의 절반 수준으로 둥글게
- 활성 칩은 반드시 체크 아이콘을 함께 사용한다.
- 비활성 칩은 `+` 기호로 추가 가능성을 표현한다.

칩은 화면을 리듬감 있게 만드는 장치다.  
행과 열이 완전히 딱 맞는 표처럼 보이기보다, 약간의 크기 차이와 위치 차이를 허용해 자연스럽게 배치한다.

### 5.6 아이콘

아이콘은 파란색 라인 아이콘을 사용한다.

권장 스타일:

- 선 굵기: 2.5px ~ 4px
- 색상: 메인 블루
- 채움 없음 또는 최소화
- 모서리가 둥근 라인 스타일
- 복잡하지 않은 단일 의미 아이콘

권장 아이콘 주제:

- 사람/타겟층
- 문서/보고서
- 시계/일정
- 돋보기/탐지
- 차트/성장
- 메일/상담
- 모바일/콘텐츠
- 데이터/네트워크
- 체크

아이콘은 장식이 아니라 의미를 보조해야 한다.  
각 카드에 아이콘을 넣을 경우 텍스트와 아이콘이 같은 개념을 말해야 한다.

### 5.7 구분선

선은 매우 제한적으로 사용한다.

사용 위치:

- 그래프 기준선
- 좌우 정보 영역 구분선
- 이미지형 장표의 제목과 본문 구분선
- 키워드 장표의 상단 문장과 칩 영역 구분선

제작 규칙:

- 얇은 선: `#D8D8D8`, 1px ~ 2px
- 강조 구분선: `#222222`, 2px
- 세로 구분선은 한 장표에 하나 이하 권장

선이 많아지면 보고서가 딱딱해지고, 템플릿의 여백 중심 미감이 약해진다.

## 6. 장표 유형별 제작 가이드

### 6.1 커버 장표

참고: `1.png`

커버는 가장 절제된 화면이다.  
중앙에 큰 제목을 두되, 화면 전체를 채우지 않고 넓은 여백으로 브랜드의 자신감을 보여준다.

구성:

- 좌상단: 파란색 날짜 캡슐
- 상단 중앙 좌측: 프로젝트 설명 문장
- 우상단: HajaCheck 로고
- 중앙 좌측: 대형 `HajaCheck` 타이틀
- 타이틀 하단: `착수보고서`
- 하단 좌측: PM 및 팀원 정보
- 하단 우측 방향: 얇은 가로선

제작 규칙:

- 날짜 캡슐은 좌측 화면 밖에서 들어온 듯한 형태로 배치한다.
- 캡슐은 너무 작으면 의미가 약하므로 충분히 넓게 둔다.
- `HajaCheck`는 화면에서 가장 큰 요소여야 한다.
- 로고는 과하게 크지 않게, 우상단에 차분히 둔다.
- 하단 인명 정보는 작지만 읽을 수 있어야 한다.
- 하단 가로선은 시각적 균형을 잡는 역할이므로 얇고 길게 사용한다.

커버에서 피해야 할 것:

- 배경 이미지 삽입
- 과도한 그래픽 장식
- 긴 문단형 설명
- 로고를 여러 번 반복
- 날짜와 타이틀의 경쟁

### 6.2 목차 장표

참고: `2.png`

목차는 큰 여백 속에 6개 섹션을 3열 x 2행으로 배치한다.  
상단의 `CONTENTS`는 파란색으로 크고 간결하게 표현한다.

구성:

- 좌상단: `CONTENTS`
- 우상단: 로고
- 하단 영역: 6개 목차 항목
- 각 항목: 큰 번호 + 굵은 제목 + 2줄 설명

제작 규칙:

- 목차 영역은 슬라이드 하단 35~40%에 배치한다.
- 번호는 제목보다 크게, 파란색으로 둔다.
- 제목은 검정 Bold, 설명은 Regular로 위계를 둔다.
- 6개 항목의 기준선과 간격을 균일하게 맞춘다.
- 설명은 항목당 최대 2줄, 가능하면 1~2개의 구문으로 제한한다.

목차는 상세 설명이 아니라 발표 흐름을 보여주는 장표다.  
각 항목에는 "무엇을 다루는지"만 남기고, 세부 내용은 본문 장표로 넘긴다.

### 6.3 데이터 비교형 장표

참고: `3.png`

두 개의 정보 덩어리를 좌우로 비교하는 유형이다.  
왼쪽은 그래프, 오른쪽은 숫자 카드처럼 서로 다른 표현 방식을 배치해 시각적 변화를 만든다.

구성:

- 상단 헤더
- 흰색 라운드 패널
- 좌측: 차트 또는 시각화
- 중앙: 세로 구분선
- 우측: 2x2 정보 카드
- 하단: 회색 설명 캡슐

제작 규칙:

- 차트는 단순해야 한다. 막대 2~4개 수준이 적절하다.
- 상승, 개선, 비교 등 핵심 메시지는 말풍선이나 화살표로 강조한다.
- 오른쪽 카드에는 큰 숫자나 짧은 키워드를 넣는다.
- 빈 카드가 있다면 실제 제작 시 반드시 내용으로 채우거나 제거한다.
- 좌우 영역의 시각적 무게가 비슷해야 한다.

차트 제작 규칙:

- 막대 색상은 메인 블루
- 보조 상승선은 라이트 블루
- 축과 눈금은 연한 회색
- 데이터 라벨은 최소화
- 차트 제목은 파란색 캡슐 라벨로 표현 가능

### 6.4 전략 흐름형 장표

참고: `4.png`

서비스 전략이나 프로세스를 설명할 때 사용하는 장표다.  
하나의 큰 흐름 안에 3개의 핵심 단계를 넣어 "하나의 플랫폼"이라는 메시지를 시각화한다.

구성:

- 상단 중앙: 핵심 문장
- 중앙: 연한 파란색 긴 라운드 트랙
- 트랙 내부: 3개의 흰색 라운드 캡슐
- 각 캡슐: 아이콘 + 단계명
- 하단: 단계별 짧은 설명
- 설명과 캡슐 사이: 세로 연결선

제작 규칙:

- 핵심 문장 중 강조어만 파란색으로 처리한다.
- 3개 단계는 같은 크기로 통일한다.
- 단계 간 간격은 일정하게 유지한다.
- 큰 트랙은 "한 플랫폼 안에서 연결된다"는 의미를 준다.
- 하단 설명은 2줄 이하로 둔다.

이 유형은 HajaCheck의 전략적 메시지에 가장 잘 맞는다.  
AI 탐지, 보고서, 상담, 검증, 산출물 관리처럼 하나의 서비스 안에 묶이는 항목을 표현할 때 활용한다.

### 6.5 이미지 + 메시지형 장표

참고: `5.png`

실제 이미지나 상징적 이미지를 크게 보여주고, 오른쪽에 핵심 메시지를 정리하는 장표다.  
이미지가 장표 분위기를 결정하므로 가장 신중하게 선택해야 한다.

구성:

- 좌측: 큰 라운드 이미지
- 이미지 우하단: 브랜드 라벨 또는 로고 버튼
- 우측: 큰 제목
- 제목 내 강조어: 파란색
- 제목 아래: 굵은 구분선
- 본문: 핵심 설명 1~2줄
- 하단: 불릿 3개

이미지 선택 기준:

- 서비스 주제와 은유적으로 연결되어야 한다.
- 지나치게 어둡거나 복잡한 이미지는 피한다.
- 텍스트와 경쟁하지 않도록 명암이 안정적이어야 한다.
- 기술 프로젝트라면 서버, 데이터, 현장 점검, 문서, 프로세스, 구조물 등 의미 있는 소재를 선택한다.

제작 규칙:

- 이미지는 패널 안에서 가장 큰 시각 요소로 둔다.
- 이미지 모서리 반경은 카드와 동일하거나 조금 크게 둔다.
- 브랜드 라벨은 이미지와 겹치되, 이미지 내용을 가리지 않아야 한다.
- 오른쪽 제목은 2줄 이내로 강하게 쓴다.
- 불릿은 3개가 가장 적절하며, 각 불릿은 한 줄 중심으로 작성한다.

### 6.6 키워드 클라우드형 장표

참고: `6.png`

주목할 미디어 키워드, 이슈 키워드, 핵심 기능 키워드 등을 시각적으로 보여주는 장표다.  
칩과 아이콘의 리듬감이 중요하다.

구성:

- 상단 중앙: 아이콘
- 중앙 상단: 기준 문장
- 얇은 가로 구분선
- 본문: 활성/비활성 키워드 칩 혼합
- 주변: 라인 아이콘

제작 규칙:

- 가장 중요한 키워드 2~3개만 활성 칩으로 둔다.
- 활성 칩은 크기를 더 크게 하거나 가운데에 배치한다.
- 비활성 칩은 보조 정보처럼 회색 테두리로 처리한다.
- 아이콘은 칩 사이 빈 공간을 메우되, 너무 많이 넣지 않는다.
- 칩은 3행 정도로 구성하면 가장 안정적이다.

키워드 문구 작성 규칙:

- 칩 하나에는 8~12자 내외가 적절하다.
- 조사나 긴 설명은 제외한다.
- "급상승 미디어 키워드"처럼 명사형으로 끝낸다.
- 활성 키워드는 실제 발표에서 강조할 내용과 일치해야 한다.

### 6.7 카드 그리드형 장표

참고: `7.png`

서비스 설계, 시장 전망, 기능 요약처럼 여러 항목을 균형 있게 보여줄 때 사용한다.

구성:

- 흰색 라운드 패널
- 내부 라이트 블루 카드 5개
- 상단 3개, 하단 2개 배치
- 카드 내부: 텍스트 좌상단 + 아이콘 우하단

제작 규칙:

- 상단 3개 카드는 같은 크기로 둔다.
- 하단 2개 카드는 가운데 정렬해 안정감을 만든다.
- 카드 간 간격은 충분히 넓게 둔다.
- 카드 텍스트는 2~3줄 이내로 제한한다.
- 아이콘 크기는 카드 너비의 20~28% 정도가 적절하다.

카드 내용 작성법:

- 제목형 문장으로 쓴다.
- 각 카드가 같은 문장 구조를 가지게 한다.
- 예: `개인화를 위한 AI 기술 도입 가속화`, `'개인 중심' 미디어로의 전환`
- 모든 카드가 동일한 수준의 개념이어야 한다.

### 6.8 결론/검증 요약형 장표

참고: `8.png`

검증 계획, 마무리, 핵심 전략, 리스크 대응처럼 정리형 메시지에 적합하다.  
왼쪽은 시각적 개념도, 오른쪽은 번호형 설명으로 구성한다.

구성:

- 좌측: 3개 원이 겹치는 벤 다이어그램
- 각 원: 핵심 키워드 + 보조 설명
- 우측: 번호형 리스트 4개
- 각 리스트: 파란 번호 + 굵은 제목 + 본문 설명

제작 규칙:

- 벤 다이어그램은 연한 블루 계열로 통일한다.
- 겹치는 영역이 너무 진해지지 않도록 투명도를 조정한다.
- 원 안 텍스트는 중앙 정렬한다.
- 오른쪽 리스트는 좌측 정렬한다.
- 번호는 파란색으로, 제목과 같은 기준선에 맞춘다.
- 본문은 2줄 이내가 이상적이며, 길어도 3줄을 넘기지 않는다.

이 장표는 발표의 결론부에서 설득력을 만든다.  
왼쪽은 개념을 한눈에 보여주고, 오른쪽은 실행 항목을 구체화하는 역할을 한다.

## 7. 정보 설계 원칙

### 7.1 한 장표에 하나의 메시지

각 슬라이드는 반드시 하나의 중심 문장을 가져야 한다.

좋은 예:

- "AI 탐지, 보고서, 상담을 하나의 플랫폼에서 제공한다"
- "수행 계획은 메뉴 오너십과 챕터 리뷰로 관리한다"
- "검증은 KPI, 산출물, 오픈 이슈 세 축으로 마무리한다"

나쁜 예:

- 프로젝트 소개, 시장 분석, 기능 정의, 일정 계획을 한 장표에 모두 넣는 것
- 제목은 전략인데 본문은 일정과 팀 역할을 설명하는 것
- 카드마다 서로 다른 수준의 정보를 섞는 것

### 7.2 시각 위계

위계는 다음 순서로 강해야 한다.

1. 섹션 번호와 장표 제목
2. 패널 안 핵심 메시지 또는 대표 시각 요소
3. 카드 제목, 활성 칩, 큰 숫자
4. 본문 설명
5. 보조 설명, 푸터

강조 요소가 너무 많으면 아무것도 강조되지 않는다.  
파란색은 한 장표에서 3~5개 지점만 강하게 사용한다.

### 7.3 텍스트 밀도

이 템플릿은 시각적으로 넓고 여유 있는 보고서 스타일이다.  
텍스트가 많아질수록 품질이 떨어져 보인다.

권장 기준:

- 제목: 1줄, 길어도 2줄
- 핵심 문장: 1~2줄
- 카드당 텍스트: 2~3줄
- 불릿: 3개
- 번호형 리스트: 4개 이하
- 한 장표 총 문장 수: 8~12개 이하

상세 설명이 많다면 발표자의 구두 설명이나 별도 부록으로 분리한다.

## 8. 그래픽 제작 규칙

### 8.1 라운드 값 통일

둥근 모서리는 템플릿의 친근하고 현대적인 인상을 만든다.

권장 반경:

- 큰 흰색 패널: 36px ~ 44px
- 라이트 블루 카드: 16px ~ 24px
- 칩/캡슐: 높이의 45% ~ 50%
- 이미지: 18px ~ 28px
- 작은 라벨: 14px ~ 20px

한 장표 안에서 반경이 제각각이면 완성도가 떨어져 보인다.  
가능하면 큰 요소, 중간 요소, 작은 요소의 3단계 반경만 사용한다.

### 8.2 그림자 사용

예시 템플릿은 그림자에 거의 의존하지 않는다.  
공간 구분은 배경색, 여백, 라운드 패널로 해결한다.

권장:

- 기본 그림자 없음
- 필요한 경우 아주 약한 그림자만 사용
- 그림자보다 색상 대비와 여백으로 계층을 만든다.

피해야 할 것:

- 카드마다 진한 그림자
- 네온 효과
- 깊은 드롭섀도
- 입체 버튼 스타일

### 8.3 차트와 도표

차트는 데이터 분석 화면이 아니라 발표용 요약 그래픽이어야 한다.

제작 규칙:

- 축, 눈금, 범례를 최소화한다.
- 핵심 변화만 보이게 한다.
- 수치가 중요하면 큰 숫자 카드와 함께 사용한다.
- 차트 색상은 블루 계열 중심으로 통일한다.
- 데이터가 많으면 표 대신 요약 카드로 바꾼다.

차트 유형별 권장:

- 전월 대비, 성장률: 막대 + 상승 화살표
- 3요소 관계: 벤 다이어그램
- 단계 흐름: 캡슐형 프로세스
- 핵심 항목 목록: 카드 그리드
- 키워드 분포: 칩 클라우드

### 8.4 이미지 사용

이미지는 드물게, 명확한 목적이 있을 때만 사용한다.  
`5.png`처럼 한 장표의 분위기를 이끌 수 있을 만큼 크게 사용해야 한다.

이미지 후보 기준:

- 실제 프로젝트 성격과 연결되는 소재
- 지나치게 연출된 스톡 이미지가 아닌 것
- 배경이 복잡하지 않은 것
- 텍스트 영역과 명확히 분리되는 것
- 회색/블루 톤과 충돌하지 않는 것

이미지 보정:

- 채도는 낮추고 명도는 안정적으로 맞춘다.
- 너무 노란색 또는 붉은색이면 색온도를 낮춘다.
- 대비가 강하면 살짝 낮춘다.
- 라운드 마스크를 적용한다.

## 9. 브랜드 적용 규칙

### 9.1 로고 사용

로고는 커버와 목차에서는 실제 로고 이미지를 사용하고, 본문 장표에서는 우상단 문서명 텍스트를 사용하는 방식이 안정적이다.

권장:

- 커버: 우상단 로고 크게 사용
- 목차: 우상단 로고 사용
- 본문: `HajaCheck 착수보고` 텍스트 사용
- 이미지형 장표: 이미지 위 작은 브랜드 라벨로 보조 사용 가능

로고 사용 시 주의:

- 로고 주변 여백을 충분히 둔다.
- 로고를 본문 강조 요소처럼 반복하지 않는다.
- 로고 색상을 임의로 바꾸지 않는다.
- 로고가 너무 작아 뭉개지면 텍스트 표기로 대체한다.

### 9.2 브랜드 톤

HajaCheck는 "하자 점검"이라는 문제 해결형 서비스 성격을 가진다.  
따라서 디자인 언어는 다음 방향을 유지한다.

- 문제를 과장하기보다 해결 구조를 명확히 보여준다.
- 탐지, 확인, 보고, 상담, 검증의 흐름을 시각적으로 반복한다.
- AI를 미래적 장식으로 표현하지 않고, 업무 효율과 정확성의 도구로 표현한다.
- 사용자는 전문가가 아니어도 이해할 수 있어야 하므로 문장은 쉽게 쓴다.

## 10. 슬라이드 제작 프로세스

### 10.1 제작 순서

1. 장표의 핵심 메시지를 한 문장으로 작성한다.
2. 메시지 유형을 고른다.
   - 비교인가?
   - 흐름인가?
   - 요약인가?
   - 키워드인가?
   - 이미지 중심인가?
3. 해당 유형의 레이아웃을 선택한다.
4. 텍스트를 먼저 최소화한다.
5. 카드, 칩, 차트, 아이콘을 배치한다.
6. 파란색 강조 지점을 3~5개로 제한한다.
7. 여백과 정렬을 맞춘다.
8. 푸터와 페이지 번호를 확인한다.
9. 전체 장표를 축소해 봤을 때 메시지가 보이는지 검수한다.

### 10.2 레이아웃 선택 기준

| 전달하려는 내용 | 권장 레이아웃 | 참고 이미지 |
|---|---|---|
| 발표 전체 구조 | 목차형 | `2.png` |
| 수치 비교, 현황 비교 | 데이터 비교형 | `3.png` |
| 서비스 전략, 프로세스 | 전략 흐름형 | `4.png` |
| 강한 메시지와 상징 이미지 | 이미지 + 메시지형 | `5.png` |
| 이슈/키워드/태그 | 키워드 클라우드형 | `6.png` |
| 여러 항목 요약 | 카드 그리드형 | `7.png` |
| 최종 정리, 검증 계획 | 결론/검증 요약형 | `8.png` |

### 10.3 문장 작성 원칙

문장은 보고서 문체를 사용하되 너무 딱딱하지 않게 쓴다.

권장 문체:

- "~합니다" 중심
- 명확한 주어와 목적어
- 짧은 문장
- 숫자와 산출물은 구체적으로
- 발표자가 설명할 수 있는 수준으로 압축

피해야 할 문체:

- 추상적인 홍보 문구
- 과장된 수식어
- 긴 복문
- 한 장표 안에서 말투가 섞이는 것
- 의미가 모호한 영어 남용

## 11. 세부 컴포넌트 사양

### 11.1 섹션 번호

- 형식: `01`, `02`, `03`
- 색상: 메인 블루
- 굵기: ExtraBold 또는 Black
- 크기: 장표 제목과 비슷하거나 약간 크게
- 정렬: 장표 제목 기준선에 맞춤

번호는 페이지 번호와 다르다.  
동일 섹션 안에서 여러 장표가 이어질 경우 같은 섹션 번호를 유지하거나, 발표 구조에 맞춰 소제목 번호를 별도로 설계한다.

### 11.2 강조어

본문 안 강조어는 파란색과 굵기만 사용한다.

좋은 예:

```text
AI 탐지 · 보고서 · 상담을 하나의 플랫폼에서 제공합니다
```

여기서 `하나의 플랫폼`만 파란색으로 강조한다.

피해야 할 강조:

- 파란색 + 밑줄 + 그림자 + 굵기 동시 사용
- 한 문장에 강조어 3개 이상
- 강조색을 여러 색으로 분산

### 11.3 불릿

불릿은 이미지형 장표와 설명형 장표에서 사용한다.

제작 규칙:

- 검정 원형 불릿 사용
- 불릿당 한 줄 권장
- 최대 3개
- 본문 크기보다 조금 작게
- 줄 간격은 여유 있게

불릿은 문장 끝을 모두 같은 방식으로 맞춘다.  
예를 들어 모두 명사형으로 끝내거나, 모두 동사형으로 끝낸다.

### 11.4 번호형 리스트

결론형 장표에서 사용하는 리스트다.

제작 규칙:

- 번호만 파란색
- 제목은 검정 Bold
- 번호와 제목은 한 줄에 배치
- 본문은 제목 아래에 배치
- 리스트 간 세로 간격은 넓게 둔다.

번호형 리스트는 실행 계획, 검증 항목, 마무리 제안처럼 순서나 항목성이 분명할 때 사용한다.

### 11.5 브랜드 라벨

이미지 위에 사용하는 작은 라벨이다.

제작 규칙:

- 파란색 라운드 배경
- 내부에 흰색 또는 로고 이미지 배치
- 이미지 우하단에 붙이되, 이미지 밖으로 일부 확장 가능
- 너무 크게 만들지 않는다.

라벨은 브랜드 인지를 보조하는 용도다.  
장표의 핵심 메시지보다 강해지면 안 된다.

## 12. 장표 간 일관성 규칙

### 12.1 고정해야 할 요소

다음 요소는 모든 본문 장표에서 동일해야 한다.

- 배경색
- 상단 헤더 위치
- 우상단 문서명 위치
- 본문 패널 위치와 크기
- 푸터 위치
- 페이지 번호 위치
- 메인 블루 색상 `#509BF8`
- 기본 폰트 에이투지체

### 12.2 변주해도 되는 요소

다음 요소는 장표 목적에 따라 변주 가능하다.

- 패널 내부 레이아웃
- 카드 개수
- 아이콘 종류
- 차트 유형
- 이미지 사용 여부
- 중앙 정렬/좌측 정렬 방식

템플릿의 완성도는 "모든 장표가 똑같은 것"이 아니라 "다른 내용을 담아도 같은 시스템처럼 보이는 것"에서 나온다.

### 12.3 장표 흐름

전체 발표자료는 다음 흐름이 자연스럽다.

1. 커버: 프로젝트와 보고서 인식
2. 목차: 발표 구조 안내
3. 프로젝트 소개: 문제와 현황
4. 서비스 전략: 해결 방향
5. 수행 계획: 작업 방식과 책임
6. 주요 키워드/환경: 외부 맥락 또는 기술 맥락
7. 서비스 설계: 핵심 구성
8. 검증 및 마무리: 산출물과 다음 단계

중간 장표를 추가하더라도 이 흐름에서 벗어나지 않도록 한다.

## 14. 자주 발생하는 문제와 해결법

### 14.1 장표가 비어 보일 때

원인:

- 핵심 시각 요소가 작다.
- 패널 내부 콘텐츠가 중앙에 모이지 않았다.
- 텍스트가 너무 짧은데 보조 그래픽이 없다.

해결:

- 대표 숫자, 아이콘, 차트, 이미지 중 하나를 크게 키운다.
- 요소 간 간격을 줄이고 하나의 덩어리로 묶는다.
- 제목 아래 핵심 문장을 추가한다.

### 14.2 장표가 복잡해 보일 때

원인:

- 텍스트가 많다.
- 카드 수가 많다.
- 파란색 강조가 과하다.
- 선과 테두리가 많다.

해결:

- 내용을 2장으로 나눈다.
- 카드 수를 3개 또는 5개로 줄인다.
- 강조색은 핵심 단어에만 남긴다.
- 구분선과 테두리를 제거하고 여백으로 분리한다.

### 14.3 브랜드 느낌이 약할 때

원인:

- 파란색 사용 규칙이 약하다.
- 체크/탐지/보고서 관련 아이콘이 없다.
- HajaCheck 문서명 위치가 흔들린다.

해결:

- 섹션 번호, 강조어, 아이콘에 메인 블루 `#509BF8`을 통일 적용한다.
- 체크, 돋보기, 문서, 데이터 아이콘을 적절히 배치한다.
- 우상단 문서명을 모든 본문 장표에 고정한다.

### 14.4 보고서답지 않고 가벼워 보일 때

원인:

- 둥근 요소가 너무 많고 크다.
- 밝은 파란색 면적이 과하다.
- 아이콘이 장식처럼 많이 들어갔다.
- 본문 근거가 부족하다.

해결:

- 큰 파란색 면은 줄이고 흰색 패널 중심으로 되돌린다.
- 아이콘 수를 줄인다.
- 검정 제목과 본문 텍스트의 위계를 강화한다.
- 구체적인 수치, 산출물명, 검증 항목을 넣는다.

## 15. 신규 장표 추가 시 추천 템플릿

### 15.1 기술 아키텍처 장표

권장 구조:

- 상단 헤더
- 흰색 패널
- 중앙: 3~4단계 아키텍처 블록
- 하단: 각 블록 설명
- 파란색 라인으로 데이터 흐름 표시

디자인 방향:

- 서버, AI, DB, 사용자 화면을 라이트 블루 카드로 표현
- 화살표는 얇고 단순하게
- 너무 많은 기술명을 한 번에 넣지 않기

### 15.2 일정 계획 장표

권장 구조:

- 흰색 패널
- 가로 타임라인
- 주차별 캡슐 또는 카드
- 주요 마일스톤만 파란색으로 강조

디자인 방향:

- 4주, 6주, 8주 단위처럼 큰 흐름으로 표현
- 세부 작업은 발표 설명으로 처리
- 위험 구간이나 리뷰 지점은 체크 아이콘으로 표시

### 15.3 산출물 목록 장표

권장 구조:

- 카드 그리드형
- 산출물 카테고리별 카드
- 카드마다 문서 아이콘 사용

디자인 방향:

- 요구사항 정의서, 화면설계서, ERD, API 명세, 테스트 계획 등
- 각 카드에는 산출물명과 목적 1줄만 넣기
- 완료/진행/예정 상태가 필요하면 작은 칩으로 표시

### 15.4 리스크 대응 장표

권장 구조:

- 좌측: 리스크 분류 카드
- 우측: 대응 전략 번호형 리스트
- 또는 2열 비교형

디자인 방향:

- 위험을 빨간색으로 과하게 표현하지 않는다.
- 파란색과 회색 중심으로 차분하게 정리한다.
- "리스크"보다 "대응 체계"가 더 크게 보이게 한다.

## 16. 최종 제작 원칙 요약

HajaCheck 착수보고 슬라이드는 **밝은 회색 배경, 흰색 라운드 패널, 포인트 컬러 `#509BF8`, 에이투지체, 굵은 제목, 넓은 여백, 라인 아이콘**이 핵심이다.

제작자는 다음 원칙을 항상 우선해야 한다.

1. 한 장표에는 하나의 메시지만 담는다.
2. 파란색은 `#509BF8`만 사용하고, 핵심을 가리키는 신호로만 사용한다.
3. 큰 흰색 패널과 넓은 여백을 유지한다.
4. 텍스트는 줄이고, 구조와 아이콘으로 이해를 돕는다.
5. 장표마다 헤더, 푸터, 페이지 번호 위치를 고정한다.
6. 기술 보고서의 신뢰감과 서비스 플랫폼의 친근함을 동시에 유지한다.

이 템플릿의 목표는 예쁘게 꾸미는 것이 아니라, **HajaCheck가 해결하려는 문제와 수행 계획을 명확하고 믿음직하게 보이게 만드는 것**이다.
